# Music System（Vue+SpringBoot前后端分离音乐系统）
软件工程开发课程项目，基于前后端分离架构实现的轻量化音乐服务系统，支持「用户端音乐浏览」与「管理员端数据管控」双场景。


## 技术栈
### 前端（用户/管理员双端界面）
- 框架：Vue 2.x
- 组件库：Element UI（实现响应式界面与表单/表格组件复用）
- 工具链：Vue Router（路由管理）、Axios（前后端接口请求）
- 构建工具：npm + webpack


### 后端（接口与数据层）
- 框架：Spring Boot 2.x
- 数据持久层：MyBatis Plus（快速实现CRUD与数据封装）
- 数据库：MySQL（存储用户、歌单、歌手等业务数据）
- 依赖管理：Maven


## 核心功能
### 1. 用户端（音乐浏览场景）
- 歌单列表展示、歌手信息查询
- 音乐基础播放交互（界面联动）


### 2. 管理员端（数据管理场景）
- 用户信息：增删改查（账号、权限配置）
- 内容管理：歌单、歌手数据的新增/编辑/删除
- 数据分页：列表类数据的分页展示与关键词查询


## 实现难点与解决思路
1. **前后端跨域与接口联调**
   开发环境下前端（8080端口）与后端（8085端口）存在跨域限制，通过在Spring Boot中配置CORS跨域规则，统一允许前端域名的请求，同时规范Axios的请求前缀与后端接口路径匹配。

2. **双端权限隔离**
   区分用户与管理员的接口访问权限，通过在后端Controller层添加权限校验逻辑，拦截非法角色的请求（如普通用户无法调用管理员的“删除用户”接口）。

3. **数据交互一致性**
   统一前后端的请求参数格式、响应数据结构（如后端返回`{code, data, msg}`的标准格式），适配前端Element UI组件的渲染要求，同时通过MyBatis Plus的`IService`/`BaseMapper`封装，减少后端重复CRUD代码。


## 使用说明
1. **前端启动**
   ```bash
   # 进入前端目录
   cd frontend
   # 安装依赖
   npm install
   # 启动开发服务
   npm run dev
   ```
   访问地址：`http://localhost:8080`


2. **后端启动**
   1. 在`application.properties`中配置MySQL数据库信息（地址、账号、密码）
   2. 启动Spring Boot主类`MusicAdminApplication.java`
   访问地址：`http://localhost:8085`
